# 2026-05-02 开发日志

## 当日主要变更

- **模型配置更新**：
  - `models.json` 默认值更新至最新提供商模型：
    - Qwen：`qwen-max` → `qwen3.5-plus`（1M 上下文，65K 输出）
    - Moonshot：`moonshot-v1-128k` → `kimi-k2.6`
    - Anthropic：`claude-3.5-sonnet` → `claude-4-sonnet-20250522`
    - OpenAI/Azure：`gpt-4-turbo`/`gpt-4` → `gpt-4o`
    - GLM-5：描述更新为 745B MoE
    - Ollama：`llama3.1` → `qwen2.5`

- **配置向导修复**：
  - 修复 `config --list` 中 `context_window` 保存 bug（`if updates:` 错误嵌套在 `backend==vllm` 块内，导致 llama-server 更新无法持久化）
  - 通用化模型文件扩展名清理：用 `Path(name).stem` 替代硬编码 `.gguf` 后缀，防止 llama-server `/v1/models` 脏 ID 覆盖 `/props` 的 `display_name`
  - 重写 `pc configure` 向导：默认分类改为国内模型；步骤顺序调整为选模型 → URL → API Key → 协议；国内模型按 `(provider, base_url)` 去重；本地模型合并 ollama+vllm 为「本地服务器」；空 Key 默认确认跳过；保存后提示运行 `pc config --list` 微调

- **编译器/语法验证（全端同步）**：
  - `MissionAdapter` / `SimpleCLI` / `TUIController` / `WebSocketManager` 统一新增修改代码文件的编译检查
  - 自动检测编译命令（gcc、g++、make、cmake、msbuild、rustc、cargo、go build、javac、npm run build、tsc），若 LLM 本轮未执行则自动补做
  - `PytestRunnerVerifier` 重命名为 `TestRunnerVerifier`，扩展支持项目级构建系统检测（make、cmake、npm、cargo、go mod、meson）
  - 支持 `skip_project_build` 参数：LLM 仍在写入文件时跳过项目构建，避免不完整源码导致误报

- **CLI 修复**：
  - 移除 `main()` 内冗余的局部 `import asyncio`，修复 `UnboundLocalError`（local variable 'asyncio' referenced before assignment）

- **测试修复**：
  - 远程模型测试中补充 mock `_probe_backend_limits`，避免每个测试发起真实 HTTP 请求并等待 18 秒网络超时
  - 测试耗时从 55 秒降至 0.33 秒

- **代码风格**：black 格式化长行元组/列表为多行（adapter/orchestrator/level2_tests/simple_cli/controller/server）

> 涉及 15+ 个文件，500+ 行变更。
