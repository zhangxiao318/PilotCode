# 2026-04-30 开发日志

## 当日主要变更

- **E2E 测试框架扩展**：
  - 新增 C 代码生成 E2E 任务（AVL 树、哈希表、JSON 解析器、内存池、环形缓冲区、文件读取器、链表、排序、栈、字符串工具）
  - 新增 `quest` 命令（`/quest`）支持批量任务下发与执行
  - WebSocket 模式支持 `auto_allow` 自动授权
  - Tokenizer 缓存优化
  - E2E Runner 新增 `--timeout` CLI 参数（默认 360 秒）
  - 超时优雅处理：捕获 `TimeoutExpired` 防止堆栈泄漏，输出 `[TIMEOUT]` 及实际耗时
  - 分析结果脚本使用 `PASS/FAIL` 替代 emoji，适配 Windows 编码

- **FileEdit 智能预编辑预览**：
  - 括号平衡差值检查（对比 old_string 与 new_string）
  - 缩进一致性检查
  - 关键结构删除检测
  - Tree-sitter 多语言语法检查（Python、JS、TS、Java、Go、Rust、C、C++、Ruby、PHP）
  - 会话级语言可用性缓存，缺失时给出一次性安装提示
  - 支持 Blocking/Warning 两级严重程度

- **SWE-bench 可靠性增强**：
  - Review Prompt 增加反删除保护
  - Explore 预算动态调整（planning 模式 12→20）
  - 放宽 Execution Prompt 的探索约束
  - Bug-fix 任务自动分类为 PLAN 模式
  - `run_headless_with_feedback` 最大轮次 3→4
  - FileEdit 范围守卫（`set_allowed_files` / `_is_file_allowed`）
  - Harness 支持 swe-env/repos 缓存复用及代理

- **Windows 兼容性与修复**：
  - PowerShell 显示、启动速度、编码回退、ANSI 清理
  - E2E Runner 子进程编码修复（utf-8 + replace），解决 Windows GBK 下中文输出 NoneType 错误
  - Windows 可执行文件解析（.exe、./  stripping）
  - 平台相关路径适配：`platformdirs`、`os.pathsep`、`os.devnull`、`tempfile.gettempdir()`、`shutil.which`
  - 危险命令/路径模式适配 Windows（`bash_tool`、`tool_sandbox`、`ai_security`、`risk_assessment`）
  - `model_client` 读取超时从 60 秒延长至 300 秒

> 涉及 30+ 个文件，3000+ 行变更。
