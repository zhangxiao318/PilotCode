【任务4：多文件联动修改】

请实现一个新的 `/timestamp` 命令，并在现有的 `/status` 命令中集成它。

具体要求：

1. 创建 `src/pilotcode/commands/timestamp_cmd.py` 文件，实现 `/timestamp` 命令：
   - 显示当前时间（格式：YYYY-MM-DD HH:MM:SS）
   - 显示项目根目录（context.cwd）
   - 显示 Python 版本（sys.version.split()[0]）
   - 命令名：timestamp，描述："Show current timestamp and environment info"，别名：ts
   - 参考其他命令文件的结构（如 version_cmd.py）

2. 修改 `src/pilotcode/commands/__init__.py`：
   - 在导入列表中添加 `from . import timestamp_cmd`（放在 version_cmd 附近即可）

3. 修改 `src/pilotcode/commands/status_cmd.py`：
   - 在 status 命令输出的末尾（return 之前），添加一个 "Last Check" 行：
   - 格式：`lines.append(f"Last Check: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")`
   - 注意：datetime 已经导入，不需要重复导入
   - 这行应该放在 "Disk Usage" 部分之后（如果存在），或放在 Token Budget 部分之后（如果不存在 Disk Usage）

4. 确保所有修改后的文件能通过 `python -m py_compile` 语法检查。

注意：
- 不要修改其他无关文件
- 保持代码风格与现有文件一致
- 新增文件的 shebang、import 顺序、async def 格式参考 version_cmd.py
